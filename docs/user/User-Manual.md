# User Manual

## Introduction

MRRG-Mobile allows field workers to access assigned jobs, report progress and communicate with the office directly from the job site.

The application is designed to remain simple to use while supporting work in areas with limited network connectivity.

The screenshots shown throughout this guide illustrate the current version of the application. Minor interface changes may occur over time while the underlying workflows remain unchanged.

---

## Logging In

Accounts are created by an administrator.

Before the first login, new users must activate their account using the activation link received by email.

After activation, the application can be accessed using the assigned email address and chosen password.

---

## Dashboard

The dashboard displays upcoming assigned work.

Jobs are grouped by day and include the information required before arriving on site, including:

- customer details;
- address;
- scheduled start time;
- job status;
- notes when available.

Workers can switch between the 3-day and weekly views depending on their planning needs.

## Completing Work

Each assigned job follows a predefined workflow.

```text
Assigned

↓

In Progress

↓

Waiting Manager Validation

↓

Completed
```

Workers update the job as work progresses.

Once the required work has been completed, the job is submitted for manager validation.

---

## Before and After Photos

Photographs provide a record of the completed work.

Before photos should be captured before work begins.

After photos should be taken once the work has been completed.

These images are attached to the job and reviewed during manager validation.

---

## Offline Operation

MRRG-Mobile continues to operate even when network connectivity is unavailable.

Previously synchronized jobs remain accessible and work performed while offline is stored locally.

Once connectivity is restored, pending changes can be synchronized with the backend.

The application indicates when displayed information originates from the local cache.

---

## Notifications

The application notifies workers when:

- new jobs are assigned;
- schedules change;
- important updates are available.

Notifications remain accessible from the notification centre even if they were previously dismissed from the device.

---

## Profile and Settings

Workers can review their profile information and access application settings from the navigation menu.

---

## Good Practices

For the best experience:

- synchronize regularly when an internet connection is available;
- capture clear before and after photographs;
- review job information before travelling to site;
- submit completed work promptly;
- keep notifications enabled to receive scheduling updates.
